Features

Everything you need in a terminal

A real SSH workspace — sessions, files, tunnels, and an optional agent bridge — wrapped in one premium desktop app.

🖥️

Real terminal

Full SSH sessions powered by russh + xterm.js — tabbed, with splittable panes, plus local shell tabs alongside your remote ones.

🗂️

Host vault

Saved hosts in folders with colours and favourites. Secrets stay in the OS keychain, and you can copy a ready-to-run ssh command per host.

🛡️

Host-key verification

First-connect fingerprint confirmation and known_hosts tracking, with a hard refusal and warning if a host key ever changes.

📁

SFTP browser

Dual-pane local/remote navigation with drag-and-drop transfer of files and whole folders, a replace/skip conflict prompt, and remote create-folder, rename, and delete.

🔀

Port forwarding & SOCKS

Local (-L) and remote (-R) forwards plus a dynamic SOCKS5 proxy (-D) — point a browser at socks5h://localhost and route through the box.

🛫

Jump hosts (ProxyJump)

Reach a host through a saved bastion in one click. The jump host's key is verified too — an unknown or changed bastion key is refused, not waved through.

📥

Import ~/.ssh/config

Pull your existing hosts — including their ProxyJump links — straight into the vault, so switching from the command line takes seconds.

📡

Broadcast input

Type once and send to every split pane in a tab — run the same command across a fleet of servers at the same time, cluster-SSH style.

🔎

Search & on-connect commands

Find text in the terminal scrollback with ⌘F, and give any host an on-connect command — tmux attach, an env setup, whatever — that runs the moment the shell opens.

🎨

Themes & settings

Community terminal themes with live controls for font size, cursor style, and scrollback — tuned to how you like to work.

🔐

Encrypted backup

Export and import your hosts and secrets to a password-encrypted file (PBKDF2 → AES-256-GCM) and restore on another machine.

⏳

Idle vault lock

An optional passphrase lock that engages after inactivity, so an unattended window doesn't leave your vault open.

🤖

AI agent access (MCP)

An optional, off-by-default MCP server bound to localhost behind a bearer token — per-host opt-in, per-command approval, and secrets never reach the agent.

Security

Built to protect your sessions

Your connections, your secrets, and your files are guarded at every step — and they stay on your machine.

Host-key & MITM protection

First-connect fingerprints are confirmed and pinned in known_hosts. If a host key ever changes, SSH Ache refuses to connect and warns you — so a man-in-the-middle can't slip in.

Secrets in the OS keychain

Passwords and keys are handed to your operating system's keychain instead of sitting in plaintext config — the platform's vault holds them, not a loose file.

Encrypted backups

Export your hosts and secrets to a single password-encrypted file using PBKDF2 key derivation and AES-256-GCM. Move it between machines without exposing anything.

Approval-gated agent bridge

The AI-agent / MCP bridge is off by default and bound to localhost behind a bearer token. Every host is opt-in and every command needs in-app approval — and your secrets never reach the agent.

Your data stays on your machine, encrypted at rest. Nothing about your hosts or sessions is sent anywhere you didn't choose.

Opening SSH Ache on macOS

SSH Ache isn't notarized yet, so on the first launch macOS shows “Apple could not verify SSH Ache is free of malware.” Use either method below — you only do this once.

1
Right-click → Open

In Applications, right-click (or Control-click) SSH Ache → Open → click Open in the dialog.

2
System Settings (macOS 15+)

Try to open the app once, then go to System Settings → Privacy & Security, scroll down to “SSH Ache was blocked” and click Open Anyway.

3
Terminal (one command)

Strip the quarantine flag, then open it normally:

xattr -dr com.apple.quarantine "/Applications/SSH Ache.app"
