SSH Ache docs

Everything you need to connect, transfer, tunnel, and keep your secrets safe.

01 Introduction

SSH Ache is a desktop SSH client with a clean terminal UI — connections, SFTP, port forwarding, and an approval-gated AI agent bridge, with your data kept on your machine and encrypted at rest.

Under the hood it is built with Tauri 2 (a Rust backend) and a React + xterm.js frontend. SSH sessions run on russh / russh-sftp, local shells on portable-pty, and your secrets are stored in the operating system keychain. The result is a single, fast, native-feeling app for macOS, Windows, and Linux.

💡

New here? Jump straight to Install, then Connecting & saving a host to open your first session.

02 Install

Homebrew (macOS)

The quickest way to install on macOS:

$ brew install --cask TanvirMahin24/sshache/sshache

Direct download

Grab a platform build — .dmg (macOS), .msi (Windows), or .AppImage / .deb (Linux) — from the GitHub Releases page.

Build from source

You'll need Rust and the Tauri prerequisites for your platform. Then, from the project root:

$ npm install
$ npm run tauri build
🛠️

Make sure the toolchain is in place first — see the Tauri prerequisites.

03 Connecting to a host & saving it

Add a host with its address, port, and username, then connect to open a full SSH session in a terminal tab. Authentication is handled with a password or key, and the secret is stored in your OS keychain rather than in plaintext.

Saved hosts live in the host vault, where you can:

  • Organise hosts into folders, give them colours, and mark favourites.
  • Open multiple sessions in tabs, and split a tab into panes to watch more than one thing at once.
  • Open a local shell tab right next to your remote sessions.
  • Copy a ready-to-run ssh command for any saved host to use elsewhere.

04 Host-key verification (MITM protection)

The first time you connect to a host, SSH Ache shows its key fingerprint and asks you to confirm it. Once accepted, the key is pinned in known_hosts and used to verify every future connection.

⚠️

If a host key ever changes, SSH Ache refuses to connect and warns you. A changed key can mean the server was rebuilt — or that someone is attempting a man-in-the-middle attack. Verify the new fingerprint out-of-band before trusting it.

05 SFTP transfers

The built-in SFTP browser gives you a dual-pane view — your local filesystem on one side, the remote host on the other — so you can move files without leaving the app.

  • Drag and drop to transfer between local and remote.
  • Move multiple files and whole folders in one go.
  • When a name already exists at the destination, a replace / skip conflict prompt lets you decide per item.

06 Port forwarding

Set up local forwards that are bridged over your SSH connection. A port on your machine is tunnelled to a destination reachable from the remote host — handy for reaching databases, dashboards, and internal services through a connection you control.

07 Themes & settings

Tune the terminal to your taste:

  • Pick from community terminal themes.
  • Adjust font size, cursor style, and scrollback live.

Changes apply immediately, so you can dial things in without restarting a session.

08 Encrypted backup (export / import)

Export your hosts and secrets to a single password-encrypted file. The file is protected with PBKDF2 key derivation and AES-256-GCM encryption, so it stays safe even if it leaves your machine.

  • Export to create a portable, encrypted snapshot of your vault.
  • Import on another machine — or after a reinstall — using the same password to restore everything.
🔑

The backup password is the only way to open the file. If you lose it, the backup can't be recovered — store it somewhere safe.

09 Idle vault lock

Turn on the optional idle vault lock to have SSH Ache require a passphrase after a period of inactivity. If you step away, an unattended window won't leave your saved hosts and secrets open.

10 AI agent access / MCP

SSH Ache can expose an optional MCP server so an AI agent can help you work across your hosts. It is designed to stay firmly under your control:

  • Off by default — nothing is exposed until you turn it on.
  • Bound to localhost and protected by a bearer token.
  • Per-host opt-in — only the hosts you allow are reachable.
  • Per-command approval — each command the agent wants to run is approved by you in the app first.
  • Your secrets never reach the agent.
🤖

Think of it as a gate, not a hand-off: the agent can ask, but every host and every command waits on your approval.

12 FAQ

Which platforms does SSH Ache run on?

macOS, Windows, and Linux. It's built with Tauri 2, so each platform gets a native build — .dmg, .msi, and .AppImage / .deb respectively.

Where are my passwords and keys stored?

In your operating system's keychain, not in a plaintext config file. SSH Ache hands secrets to the platform's secure store and retrieves them when a connection needs them.

Does my data leave my machine?

Your data stays on your machine, encrypted at rest. Backups you export are password-encrypted files that only move if you move them, and the optional agent bridge is bound to localhost behind approvals.

Is the AI agent bridge safe to enable?

It's off by default. When enabled, it binds to localhost behind a bearer token, is opt-in per host, requires per-command approval in the app, and never receives your secrets. You stay in control of every action.

What happens if a server's host key changes?

SSH Ache refuses to connect and warns you. Confirm the new fingerprint through a trusted channel before accepting it — a changed key can indicate a man-in-the-middle attempt.

How do I move my setup to a new computer?

Use the encrypted backup. Export your hosts and secrets to a password-encrypted file on the old machine, then import it with the same password on the new one.
